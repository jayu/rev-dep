# Running checks

Run all enabled checks:

```bash
rev-dep config run
```

List every issue instead of the default truncated output:

```bash
rev-dep config run --list-all-issues
```

## What to expect

- Exit code `0` when all enabled checks pass
- Exit code `1` when failures exist
- Human-readable output by default
- JSON output available via `--format json`

## Useful flags

```bash
rev-dep config run --fix
rev-dep config run --fix --recheck
rev-dep config run --rules packages/app,packages/shared
rev-dep config run --format issues-list
```

- `--fix`: apply fixes for fixable issues.
- `--recheck`: after fixing, run the checks again and report the post-fix state.
- `--rules`: run only selected rule paths from the config.
- `--format issues-list`: print a compact issue list grouped by issue type.

## Output behavior details

- Human output truncates each issue section by default.
- `--list-all-issues` removes that truncation.
- `--format json` and `--format issues-list` both force detailed processing suitable for machine-friendly or compact issue review.

## Exit code nuance with `--fix`

Without `--fix`, any failure makes the command exit non-zero.

With `--fix`, the command still exits non-zero if unfixable issues remain after the run. In other words, a successful fix pass is not the same as a clean repository.
