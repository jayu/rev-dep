# Rev-dep

`rev-dep` is a fast dependency analysis CLI for JavaScript and TypeScript projects. It helps you enforce architecture rules, find dead code, trace dependency paths, and inspect package usage without stitching together multiple slower tools.

There are two main ways to use it:

- `rev-dep config run` for repeatable codebase governance in CI and local checks.
- Standalone CLI commands for ad hoc investigation while debugging or refactoring.

## What Rev-dep Helps You Find

- Illegal imports across architectural boundaries
- Circular imports
- Unused exports and orphan files
- Missing or unused `node_modules`
- Imports that no longer resolve
- Dependency paths between files and entry points

## Core Ideas

- Rev-dep builds a dependency graph from source files.
- Entry points determine what counts as reachable production or development code.
- Config-based checks are the preferred path when you want one fast pass over the whole project.
- Exploratory commands are better when you want to answer one question quickly.

If the terms above are unfamiliar, start with [Glossary](./glossary).
