# FAQ

## Why is a package marked unused when we definitely need it?

Because `rev-dep` is looking for package usage in the processed source graph. Tooling-only dependencies may not appear there.

## Why does orphan detection look wrong?

Usually because entry points are incomplete or workspace-package traversal is not modeled the way your repo actually works.

## Should I use standalone CLI commands or config run?

Use standalone commands for exploration. Use `config run` for repeatable governance and CI.

## Should I follow all workspace packages?

Only when that matches the analysis scope you want. Broader graphs can change reachability results.
