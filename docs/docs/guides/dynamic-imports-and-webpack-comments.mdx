# Dynamic imports and webpack magic comments

Rev-dep supports common dynamic import forms, including webpack magic comments between `import(` and the module string.

## Why this page exists

Dynamic imports are harder to parse than static imports, and malformed or unusual patterns can lead to parser failures or missing edges.

## Practical advice

- prefer conventional `import("...")` forms where possible
- keep an eye on unusual generated or minified source
- if a dynamic import appears unresolved, reduce it to a small repro before assuming the whole graph is wrong
